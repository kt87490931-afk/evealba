# 이브알바 실시간 채팅 구축 액션플랜

## 개요
- **목표**: 스코어포인트 채팅 플러그인(`plugin/chat`)을 이브알바에 맞게 개조하여 지역별 실시간 채팅 구현
- **이용 대상**: 일반회원(여성, `mb_1='normal'` AND `mb_sex='F'`) 전용
- **위치**: 메인페이지 우측 플로팅 버튼 → 채팅창 열기/닫기
- **디자인 기준**: `eve_alba_chat_1.html` (핑크 테마, 지역 드롭다운, 👩 아바타)
- **서버**: 현재 서버(188.166.179.115)에 동일 배치 (분리 불필요)

## 데이터베이스 구조

### 재사용 테이블 (ScorePoint → 이브알바)
| 테이블 | 용도 | 변경사항 |
|--------|------|----------|
| `g5_chat_msg` | 채팅 메시지 | `cm_region VARCHAR(10)` 컬럼 추가 |
| `g5_chat_config` | 채팅 설정 | 그대로 사용 |
| `g5_chat_ban` | 밴 목록 | 그대로 사용 |
| `g5_chat_online` | 접속자 추적 | `co_region VARCHAR(10)` 컬럼 추가 |
| `g5_chat_report` | 신고 기록 | 그대로 사용 |

### ALTER 문 (1단계에서 실행)
```sql
ALTER TABLE g5_chat_msg ADD COLUMN cm_region VARCHAR(10) NOT NULL DEFAULT '전체' AFTER cm_content;
ALTER TABLE g5_chat_msg ADD KEY idx_region (cm_region);

ALTER TABLE g5_chat_online ADD COLUMN co_region VARCHAR(10) NOT NULL DEFAULT '전체';
```

## 회원 데이터 매핑 (기존 가입 필드 활용)
| 필드 | 용도 | 값 |
|------|------|-----|
| `mb_1` | 회원유형 | `normal`(일반) / `biz`(기업) |
| `mb_sex` | 성별 | `M`(남) / `F`(여) |
| `mb_nick` | 닉네임 | 채팅 표시명 |
| `mb_id` | 아이디 | 밴/신고 식별자 |

### 채팅 권한 판단
```php
$can_chat = ($is_member && $member['mb_1'] === 'normal' && $member['mb_sex'] === 'F');
```

## 단계별 액션플랜

### 1단계: 플러그인 설치 및 DB 준비
**작업 내용:**
1. `plugin/chat` 폴더를 이브알바 프로젝트(`d:\evealba\plugin\chat\`)에 복사
2. `_common.php` 수정 — ScorePoint 전용 코드 제거
3. `install.php` 수정 — `cm_region`, `co_region` 컬럼 포함
4. 서버에서 `install.php` 실행하여 테이블 생성

**완료 기준:** DB 테이블 4개 생성 확인

### 2단계: chat_ajax.php 개조 (백엔드 핵심)
**작업 내용:**
1. ScorePoint 전용 코드 제거:
   - `sp_chat_level_from_member()` → 제거
   - `sp_chat_get_icon_url()` → `👩` 고정 반환
   - `sp_badge_*` 뱃지 코드 → 제거
   - ScorePoint theme 경로 참조 → 제거
2. 채팅 권한 체크 추가:
   - `send` 액션: `mb_1 === 'normal'` AND `mb_sex === 'F'` 확인
   - 기업회원/남성은 "일반회원(여성)만 채팅이 가능합니다." 에러 반환
3. 지역 기능 추가:
   - `send`: `$_POST['region']` 파라미터 받아서 `cm_region`에 저장
   - `list`: `$_GET['region']` 파라미터로 필터 (`전체`이면 WHERE 없음)
   - `ping`: 지역 정보 포함하여 접속자 카운트
4. 지역별 접속자 수 반환:
   - `hello`/`list` 응답에 `region_counts` 배열 추가

**수정 파일:** `plugin/chat/chat_ajax.php`

**완료 기준:** curl로 send/list/hello 테스트 정상

### 3단계: 프론트엔드 (chat_box.php → eve_chat_box.php)
**작업 내용:**
1. `chat_box.php`를 `eve_chat_box.php`로 복사 후 전면 교체
2. `eve_alba_chat_1.html`의 디자인을 PHP 템플릿으로 변환:
   - 헤더: 💬 실시간 채팅 + 지역 드롭다운(전체 ▼) + 접속자 수
   - 지역 선택: 전체/서울/경기/인천/부산/대구/광주/대전/울산/강원/경남/경북/전남/세종/제주
   - 메시지 영역: 버블 형태(내 메시지 우측, 상대방 좌측)
   - 공지 배너 (관리자 설정 가능)
   - 입력창: Enter 전송, Shift+Enter 줄바꿈
   - 채팅규정 모달
   - 무시목록 모달
3. JavaScript 핵심 로직:
   - AJAX 폴링 (4초 간격, list 호출)
   - 지역 변경 시 필터 적용
   - 메시지 전송 시 현재 선택 지역 함께 전송
   - 10분 무활동 시 연결 중단 (기존 로직 유지)
   - 로그인 안 했으면 "로그인 후 이용" 안내
   - 남성/기업회원이면 "일반회원(여성)만 이용 가능" 안내
4. 닉 클릭 메뉴: 무시하기, 신고하기 (기존 로직 간소화)

**수정 파일:** `plugin/chat/eve_chat_box.php` (신규)

**완료 기준:** 브라우저에서 채팅 송수신 정상

### 4단계: 플로팅 버튼 연동
**작업 내용:**
1. 메인 레이아웃(`tail.sub.php` 또는 `index_main.php`)에 플로팅 버튼 영역 확인
2. 기존 3개 플로팅 버튼(카카오톡/채팅/위로) 중 채팅 버튼에 `onclick="toggleEveChat()"` 연결
3. 채팅창을 `position:fixed; bottom:90px; right:28px;`로 배치
4. 열기/닫기 토글 + 닫혔을 때 새 메시지 뱃지 표시
5. `eve_chat_box.php`를 `include_once`로 삽입

**수정 파일:** 플로팅 버튼 위치의 레이아웃 파일

**완료 기준:** 플로팅 버튼 클릭 → 채팅창 열림/닫힘

### 5단계: 관리자 기능
**작업 내용:**
1. 채팅 관리 페이지 (`adm/chat_manage.php`) 생성 또는 기존 `chat_admin.php` 활용
2. 관리 기능:
   - 채팅 동결/해제
   - 회원 밴 (닉네임 기반, 시간제/영구)
   - 공지 수정
   - 채팅규정 수정
   - 금칙어 관리
   - 도배/반복 제한 설정
   - 채팅 전체삭제
   - 신고 목록 확인
3. 관리자는 채팅창에서 🛡 아이콘으로 관리 페이지 접근

**수정 파일:** `plugin/chat/chat_admin.php` 또는 별도 생성

**완료 기준:** 관리자 로그인 → 채팅 설정 변경 가능

### 6단계: 테스트 및 배포
**작업 내용:**
1. 테스트 시나리오:
   - [ ] 일반회원(여성) 로그인 → 채팅 가능
   - [ ] 일반회원(남성) 로그인 → 채팅 불가 메시지
   - [ ] 기업회원 로그인 → 채팅 불가 메시지

   - [ ] 금칙어 포함 → 전송 차단
   - [ ] 도배 → 연속 전송 제한
   - [ ] 밴된 회원 → 전송 불가
   - [ ] 신고하기 → 접수 확인
   - [ ] 무시하기 → 해당 회원 메시지 숨김
   - [ ] 10분 무활동 → 연결 중단, 클릭 시 재연결
   - [ ] 관리자 동결 → 전체 입력 잠금
   - [ ] 플로팅 버튼 → 채팅창 열기/닫기

## 주의사항 및 실수 방지

### 흔한 실수와 해결책

| 실수 | 원인 | 해결 |
|------|------|------|
| `common.php` include 실패 | 경로 차이 (`plugin/chat` → 2단계 위) | `__DIR__ . '/../../common.php'` 사용 |
| 테이블 없음 에러 | install.php 미실행 | SSH 접속 후 브라우저에서 install.php 실행 |
| 채팅 메시지 안 나옴 | `last_id`가 최신이라 이전 메시지 미표시 | `hello` 후 `last_id=0`으로 초기 로드 |
| CORS 에러 | 도메인 불일치 | 동일 서버이므로 해당 없음 |
| 한글 깨짐 | charset 불일치 | `utf8mb4` 통일 확인 |
| 폴링 부하 | 접속자 과다 시 | 비활성 탭 간격 늘리기 (10초) |
| 세션 만료 | 장시간 채팅 | 10분 무활동 중단 로직 유지 |
| 동일 mb_id로 밴 중복 | ON DUPLICATE KEY | 기존 로직 그대로 (안전) |

### 절대 하지 말아야 할 것
1. ScorePoint 테마 경로(`theme/scorepoint`) 참조 남기지 않기
2. `g5_member` 테이블 스키마 변경하지 않기
3. 기존 이브알바 기능(채용정보, 이력서 등)에 영향 주지 않기
4. `chat_ajax.php`에서 관리자 액션을 비로그인 상태에서 호출 가능하게 두지 않기
