# 쪽지관리 상세 구현 플랜 (이브알바)

## 1. 개요

이브알바에서 쪽지는 **알람/알림** 기능으로 활용됩니다. 어드민에서 전체 쪽지 발송, 회원가입 시 자동 쪽지, 쿠폰 발송 시 쪽지 동반 기능을 구현합니다. 추후 기프티콘 발송 시에도 동일한 대상 구분을 사용합니다.

### 1.1 대상 구분

| 대상 코드 | 설명 | SQL 조건 |
|-----------|------|----------|
| `general` | 일반회원 | `(mb_1 = '' OR mb_1 != 'biz')` AND `mb_leave_date = ''` AND `mb_intercept_date = ''` |
| `biz` | 기업회원 | `mb_1 = 'biz'` |
| `biz_approved` | 승인된 기업회원 | `mb_1 = 'biz'` AND `mb_7 = 'approved'` |
| `all` | 전체 회원 | 탈퇴·차단 제외 전체 |

### 1.2 기능 목록

| 기능 | 설명 | 구현 우선순위 |
|------|------|---------------|
| 전체 쪽지 보내기 | 대상(일반회원/기업회원) 선택 후 일괄 발송 | 1 |
| 회원가입 시 자동 쪽지 | 설정 on/off, 대상별 환영 메시지 | 2 |
| 전체 쿠폰 발송 시 쪽지 동반 | 쿠폰 일괄 발급 시 알림 쪽지 옵션 | 3 |

## 2. 전체 흐름

### 2.1 전체 쪽지 보내기 흐름

```
1. 어드민: 이브알바 전용 → 쪽지관리 메뉴 클릭
2. 전체 쪽지 보내기 폼:
   - 대상 선택: [일반회원] [기업회원(승인)] [전체회원]
   - 내용 입력 (65536자 제한)
3. 발송 버튼 → 확인 팝업
4. 서버: 대상별 mb_id 목록 SELECT → memo_table INSERT (recv+send 각 1행)
5. mb_memo_call, mb_memo_cnt 업데이트
6. 결과: "N명에게 쪽지를 발송했습니다."
```

### 2.2 회원가입 시 자동 쪽지 흐름

```
1. 회원가입 완료 (register_form_update.php)
2. run_event('register_form_update_after', ...) 호출
3. extend 또는 lib의 훅에서:
   - 설정 확인: em_join_memo_on = 1 ?
   - 대상 확인: 신규회원이 일반/기업 중 어떤 유형인지
   - 해당 설정값에 따라 환영 쪽지 INSERT
```

### 2.3 전체 쿠폰 발송 시 쪽지 동반 흐름

```
1. 어드민: 쿠폰 관리 → 개별 쿠폰 발급 페이지 (eve_coupon_issue.php)
2. 일괄 발급 폼에 [발급 시 쪽지 동시 발송] 체크박스 추가
3. bulk 처리 시:
   - 기존: g5_ev_coupon_issue INSERT
   - 추가: 체크 시 발급된 mb_id 목록에 대해 쪽지 INSERT
```

## Part 1: DB · 설정

### 1.1 목표

- 쪽지관리 설정 저장용 테이블 또는 g5_config 확장
- 회원가입 시 자동 쪽지 on/off 및 메시지 템플릿

### 1.2 테이블 설계

**신규 테이블: `g5_ev_memo_config`**

```sql
CREATE TABLE IF NOT EXISTS g5_ev_memo_config (
  emc_id TINYINT UNSIGNED NOT NULL DEFAULT 1 PRIMARY KEY,
  em_join_memo_on TINYINT(1) NOT NULL DEFAULT 0 COMMENT '회원가입 시 자동 쪽지 0:off 1:on',
  em_join_memo_general TEXT DEFAULT NULL COMMENT '일반회원 가입 시 쪽지 내용',
  em_join_memo_biz TEXT DEFAULT NULL COMMENT '기업회원 가입 시 쪽지 내용',
  em_updated DATETIME DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

INSERT INTO g5_ev_memo_config (emc_id, em_join_memo_on, em_join_memo_general, em_join_memo_biz)
VALUES (1, 0, '이브알바에 가입해 주셔서 감사합니다.', '이브알바에 기업회원으로 가입해 주셔서 감사합니다. 승인 후 서비스를 이용하실 수 있습니다.')
ON DUPLICATE KEY UPDATE emc_id=emc_id;
```

### 1.3 마이그레이션 파일

- **경로**: `adm/run_migration_016.php` (기존 015 다음)
- **내용**: 위 CREATE TABLE + INSERT 실행
- **실행 후**: 파일명 로그 또는 `data/` 에 완료 플래그 기록

### 1.4 해결책 (실패 시)

- `g5_ev_memo_config`가 이미 존재하면 CREATE IF NOT EXISTS로 오류 없음
- INSERT 시 ON DUPLICATE KEY UPDATE로 중복 무시
- 같은 실수 재발 시: 마이그레이션 파일 상단에 `if (table_exists) return;` 로 이중 실행 방지

- `action=config_save` (POST): 설정 저장 → `g5_ev_memo_config` UPDATE

### 2.4 전체 쪽지 발송 로직

**대상별 SQL**:

| target | WHERE 조건 |
|--------|------------|
| `general` | `(mb_1 = '' OR mb_1 != 'biz') AND (mb_leave_date = '' OR mb_leave_date IS NULL) AND (mb_intercept_date = '' OR mb_intercept_date IS NULL)` |
| `biz_approved` | `mb_1 = 'biz' AND mb_7 = 'approved' AND (mb_leave_date = '' OR mb_leave_date IS NULL) AND (mb_intercept_date = '' OR mb_intercept_date IS NULL)` |
| `all` | `(mb_leave_date = '' OR mb_leave_date IS NULL) AND (mb_intercept_date = '' OR mb_intercept_date IS NULL)` |

**발송 프로세스** (bbs/memo_form_update.php 로직 참고):
1. `me_memo` 최대 65536자, sql_escape
2. 관리자 mb_id를 me_send_mb_id로 사용
3. 각 수신자마다: recv 1행 INSERT → me_id 획득 → send 1행 INSERT
4. `mb_memo_call`, `mb_memo_cnt` 업데이트
5. insert_point 호출하지 않음 (관리자 발송이므로)

**청크 처리** (회원 수 1000명 이상 대비):
- PHP `max_execution_time` 증가 또는 `set_time_limit(300)`
- 500명 단위로 처리 후 flush (선택)
- 또는 AJAX로 청크별 요청 (우선순위 낮음, 1차는 동기 처리)

### 2.5 해결책 (실패 시)

- `me_id` 중복: `select max(me_id)+1` 대신 `sql_insert_id()` 사용 (memo_form_update.php 방식 유지)
- 대량 발송 타임아웃: `set_time_limit(300)` 적용, 필요 시 청크 분할
- mb_1이 NULL인 경우: `(mb_1 = '' OR mb_1 IS NULL OR mb_1 != 'biz')` 로 일반회원 판별

## Part 3: 회원가입 시 자동 쪽지

### 3.1 목표

- `register_form_update_after` 훅에서 자동 쪽지 발송
- 설정 on/off, 일반·기업 회원별 메시지 적용

### 3.2 훅 파일

**파일**: `extend/ev_memo_register.extend.php` (신규)  
※ extend 폴더는 `*.php` 파일만 자동 로드됨

**로직**:
```php
// register_form_update_after 훅 등록
// 1. g5_ev_memo_config 조회
// 2. em_join_memo_on != 1 이면 return
// 3. mb_id로 회원 조회 → mb_1 확인
// 4. mb_1 == 'biz' → em_join_memo_biz, 아니면 em_join_memo_general
// 5. 메시지 비어있으면 기본 문구 사용
// 6. ev_send_memo($recv_mb_id, $content, $send_mb_id='admin') 호출
```

### 3.3 공통 쪽지 발송 함수

**파일**: `lib/ev_memo.lib.php` (신규)

**함수**: `ev_send_memo($recv_mb_id, $content, $send_mb_id = '')`

- `$send_mb_id` 비어있으면 관리자(level 10) mb_id 사용
- `$content` 65536자 제한
- memo_form_update.php와 동일한 INSERT 로직 (recv + send)
- mb_memo_call, mb_memo_cnt 업데이트
- 포인트 차감 없음

**용도**: 전체 쪽지, 회원가입 자동 쪽지, 쿠폰 발송 시 쪽지 등 공통 사용

### 3.4 훅 등록

**파일**: `extend/ev_memo_register.extend.php` 내부에서 add_event 호출

- 그누보드 훅: `add_event('register_form_update_after', 'ev_memo_on_register_after', 10, 2);`
- `ev_memo_on_register_after($mb_id, $w)` 함수 정의
- `$w == ''` 일 때만 신규 가입으로 처리 (수정 시 제외)

### 3.5 extend 로드 확인

- `common.php` 또는 `config.php`에서 `extend/` 경로 include 여부 확인
- 없다면 `data/extend/` 등 기존 패턴에 맞춰 include 추가

### 3.6 해결책 (실패 시)

- 훅이 호출되지 않음: hook.lib.php의 add_event/run_event 동작 확인, 함수명 오타 확인
- 기업회원인데 일반 메시지 발송: mb_1 체크 순서 확인 (biz 우선)
- 중복 발송: `$w == ''` 조건으로 신규 가입만 처리

## Part 4: 쿠폰 발송 시 쪽지 동반

### 4.1 목표

- `eve_coupon_issue.php` 일괄 발급 시 "발급 시 쪽지 동시 발송" 옵션 추가
- 발급된 회원에게 "쿠폰이 발급되었습니다" 안내 쪽지

### 4.2 수정 파일

**파일**: `adm/eve_coupon_issue.php`

**변경 사항**:
1. 일괄 발급 폼에 체크박스 추가: `name="send_memo" value="1"`
2. `action=bulk` 처리 블록 내:
   - `$send_memo = isset($_POST['send_memo']) && $_POST['send_memo'] == '1';`
   - bulk 처리 후 `$issued_mb_ids` 배열에 발급된 mb_id 수집
   - `$send_memo`이면 `ev_send_memo($mb_id, $memo_content)` 호출
3. 쪽지 내용: `"{쿠폰명} 쿠폰이 발급되었습니다. 마이페이지에서 확인해 주세요."` (또는 설정 가능하게)

### 4.3 쪽지 내용 템플릿

- 기본: `"{$ec['ec_name']} 쿠폰이 발급되었습니다. 마이페이지에서 확인해 주세요."`
- 추후 기프티콘 발송 시에도 동일 패턴으로 확장

### 4.4 해결책 (실패 시)

- 발급 실패 시 쪽지 발송 안 함: bulk 루프 내에서 INSERT 성공한 경우에만 mb_id를 배열에 추가
- 쿠폰명 XSS: `htmlspecialchars` 또는 `get_text` 적용

## Part 5: 기프티콘 발송 대비 (추후)

### 5.1 확장 포인트

- 전체 쪽지/쿠폰 발송 시 **대상 선택 UI**를 `general` / `biz_approved` 로 통일
- 기프티콘 발송 화면에서도 동일한 대상 선택 컴포넌트 재사용
- `lib/ev_memo.lib.php`의 `ev_send_memo()` 가 모든 알림 발송의 공통 함수로 활용

### 5.2 대상 코드 일관성

| 기능 | 대상 사용 |
|------|----------|
| 전체 쪽지 | general, biz_approved, all |
| 쿠폰 bulk | biz_approved (기존) + 일반회원용 쿠폰 시 general |
| 기프티콘 bulk (추후) | general 우선 (ec_target=general) |
