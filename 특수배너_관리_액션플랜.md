# 특수배너 관리 액션플랜 (히어로배너 · 추천업소 플로팅)

## 1. 개요

특수배너는 **히어로배너**와 **추천업소(플로팅배너)** 2종류로, 자리 수가 제한되어 있어 고객센터 문의 → 계약 → 입금 방식으로만 등록 가능합니다. 어드민 전용 "특수배너 관리" 페이지에서 일괄 관리합니다.

### 1.1 특수배너 종류

| 배너 | 위치 | 최대 수량 | 관리 주체 | 비고 |
|------|------|----------|----------|------|
| **히어로배너** | 메인 상단 대형 배너 영역 | 2개 (예시) | 어드민 직접 생성/편집 | 썸네일 에디터 모든 옵션 사용 |
| **추천업소(플로팅)** | 우측 플로팅 패널 | 6개 | 어드민 승인 → 기업회원 기존 관리 | 기존 jobs_ongoing.php에서 관리 |

### 1.2 일반 광고와의 차이

| 구분 | 일반 광고 (줄광고~추천) | 특수배너 |
|------|----------------------|---------|
| **수량** | 무제한 | 자리 수 제한 |
| **가격** | 고정가 (기간별 30/60/90일) | 협의가 (자리 희소성) |
| **결제** | jobs_register.php에서 셀프 체크 → 입금 | 고객센터 문의 → 자리 확인 → 협의 → 입금 |
| **시스템** | 자동화 | 반드시 어드민 개입 |

### 1.3 배포 규칙

- **각 Part 완료 시 반드시 자동 배포** (Git push + 서버 SCP 또는 deploy.ps1 -PushToServer)

## 2. 전체 흐름

### 2.1 추천업소(플로팅) 등록 흐름

```
1. 고객: 고객센터 문의 "추천업소 플로팅배너 하고 싶습니다"
2. 어드민: "특수배너 관리" → 현황 확인 (예: 추천업소 5/6 → 1자리 가능)
3. 어드민: 가격 안내 + 계약 협의
4. 고객: 입금 (프리미엄+우대+줄광고+특수배너 합산 금액)
5. 고객: jobs_register.php에서 채용정보 작성 (프리미엄+우대+줄광고 체크)
6. AI글 생성 (기존 흐름 동일)
7. 어드민: 입금확인 → "특수배너 관리"에서 해당 jr_id를 추천업소로 승인
   → jr_ad_labels에 "추천업소" 자동 추가
8. 플로팅 패널에 해당 광고 노출
9. 기업회원: 기존 jobs_ongoing.php에서 썸네일/연장/점프 등 관리 (추가 개발 없음)
```

### 2.2 히어로배너 등록 흐름

```
1. 고객: 고객센터 문의 "히어로배너 하고 싶습니다"
2. 어드민: "특수배너 관리" → 히어로 현황 확인 (예: 1/2 → 1자리 가능)
3. 어드민: 가격 안내 + 계약 협의
4. 고객: 입금
5. 어드민: "특수배너 관리 → 히어로배너" 에서 직접 생성
   - 썸네일 에디터 (jobs_view와 동일한 모든 옵션)
   - 링크 URL (클릭 시 이동), 텍스트, 기간 설정
6. 메인 상단 히어로 영역에 노출
```

## Part 1: DB 설계

### 1.1 목표

- 히어로배너 전용 테이블 생성 (추천업소는 기존 g5_jobs_register 활용)

### 1.2 히어로배너 테이블

```sql
CREATE TABLE IF NOT EXISTS g5_special_banner (
  sb_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  sb_type ENUM('hero') NOT NULL DEFAULT 'hero' COMMENT '배너 유형',
  sb_status ENUM('active','inactive','expired') NOT NULL DEFAULT 'inactive' COMMENT '상태',
  sb_position TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '표시 순서',
  sb_mb_id VARCHAR(20) NOT NULL DEFAULT '' COMMENT '연결 기업회원 ID (참고용)',
  sb_jr_id INT UNSIGNED DEFAULT NULL COMMENT '연결 채용광고 ID (참고용)',
  sb_link VARCHAR(500) NOT NULL DEFAULT '' COMMENT '클릭 시 이동 URL',
  sb_start_date DATE DEFAULT NULL COMMENT '노출 시작일',
  sb_end_date DATE DEFAULT NULL COMMENT '노출 종료일',
  sb_data JSON DEFAULT NULL COMMENT '썸네일 옵션 JSON (gradient, title, text, icon, motion, wave, border 등)',
  sb_memo TEXT DEFAULT NULL COMMENT '어드민 메모 (계약 내용, 금액 등)',
  sb_created DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  sb_updated DATETIME DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_type_status (sb_type, sb_status),
  KEY idx_position (sb_position)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 1.3 추천업소는 별도 테이블 불필요

- 기존 `g5_jobs_register.jr_ad_labels`에 "추천업소" 라벨 추가/제거로 처리
- `get_jobs_by_type('추천업소')` 로 조회 가능 (기존 헬퍼 그대로 활용)

### 1.4 해결책 (실패 시)

- JSON 컬럼 미지원 시: `sb_data`를 `LONGTEXT`로 변경, PHP에서 json_encode/decode 처리
- 추천업소 라벨이 기존 "추천"과 충돌 시: "특수_추천업소" 등 고유 라벨 사용

## Part 4: 어드민 - 추천업소(플로팅) 승인/해제

### 4.1 목표

- 기존 ongoing 채용광고 중 추천업소로 승인/해제
- `jr_ad_labels`에 "추천업소" 라벨 추가/제거

### 4.2 승인 흐름

```
1. "추천업소 승인 추가" 클릭
2. 현재 ongoing인 채용광고 목록 모달/페이지 표시
   - 검색: jr_id, 업소명, 회원ID
3. 선택 → 승인
   - jr_ad_labels에 "추천업소" 추가 (기존 라벨 유지)
   - 승인 시각 기록 (메모 또는 별도 필드)
4. 해제 시
   - jr_ad_labels에서 "추천업소" 제거
```

### 4.3 액션 항목

| 순번 | 액션 | 파일 | 산출물 |
|------|------|------|--------|
| 4.1 | 추천업소 승인 UI: ongoing 채용광고 검색 + 선택 | adm/eve_special_banner.php | 승인 UI |
| 4.2 | 승인 처리: jr_ad_labels에 "추천업소" 추가 + 순서(position) 설정 | adm/eve_special_banner_recommend_update.php | 승인 API |
| 4.3 | 해제 처리: jr_ad_labels에서 "추천업소" 제거 | adm/eve_special_banner_recommend_update.php | 해제 API |
| 4.4 | 순서 변경: 드래그 또는 순번 입력으로 플로팅 패널 표시 순서 변경 | adm/eve_special_banner.php | 순서 변경 UI |
| 4.5 | 최대 수량 체크 (승인 시 현재 추천업소 수 < 6) | adm/eve_special_banner_recommend_update.php | 수량 제한 |

### 4.4 해결책 (실패 시)

- jr_ad_labels 파싱 오류: 쉼표 구분 문자열이므로, 추가 시 중복 체크, 제거 시 빈 쉼표 정리
- 순서 관리: jr_ad_labels만으로는 순서 저장 불가 → 별도 `g5_special_banner_order` 테이블 또는 `g5_special_banner`에 recommend 타입 행 추가
- 채용광고 만료 시 자동 해제: 크론에서 ongoing이 아닌 추천업소 라벨 자동 제거

## Part 5: 프론트엔드 DB 연동 (하드코딩 → DB 조회)

### 5.1 목표

### 5.2 액션 항목

| 순번 | 액션 | 파일 | 산출물 |
|------|------|------|--------|
| 5.1 | 히어로배너 렌더링: `g5_special_banner` WHERE sb_type='hero' AND sb_status='active' AND sb_start_date<=TODAY AND sb_end_date>=TODAY 조회 → 기존 하드코딩 교체 | theme/evealba/inc/ads_main_banner.php | DB 연동 히어로배너 |
| 5.2 | 히어로배너 썸네일 옵션 적용: sb_data JSON에서 gradient, title, text, icon, motion, wave 등 파싱 → 렌더링 | theme/evealba/inc/ads_main_banner.php | 동적 렌더링 |
| 5.3 | 추천업소 플로팅 렌더링: `get_jobs_by_type('추천업소', 6)` 조회 → tail.php 하드코딩 교체 | theme/evealba/tail.php | DB 연동 플로팅 |
| 5.4 | 추천업소 카드 렌더링 함수 생성 또는 기존 render 함수 활용 | extend/jobs_list_helper.php 또는 tail.php | 렌더링 함수 |

### 5.3 해결책 (실패 시)

- DB 조회 실패 시: 기존 하드코딩 데이터를 fallback으로 유지
- 히어로배너 렌더링 깨짐: sb_data 키가 기존 jobs_view 에디터와 동일하므로, 기존 렌더링 함수 재사용
- 추천업소 정렬: `get_jobs_by_type` 기본 정렬(점프순)과 별도로, 어드민이 설정한 순서 적용 필요 시 ORDER BY 추가

## 9. 실수했을 때의 해결책

### 추천업소 라벨 파싱 오류
- **원인**: jr_ad_labels가 "줄광고,우대,프리미엄,추천업소" 형태인데 공백/중복 문제
- **해결**: 추가 시 trim + 중복 체크, 제거 시 빈 쉼표(,,) 정리, 앞뒤 쉼표 제거

### 히어로배너 최대 수량 초과
- **원인**: 동시에 2명의 어드민이 추가
- **해결**: INSERT 전 COUNT 재확인 (트랜잭션)

### 추천업소 채용광고 만료 후 플로팅에 계속 노출
- **원인**: 크론 미실행 또는 라벨 제거 실패
- **해결**: 프론트 조회 시 `jr_status='ongoing' AND jr_end_date>=CURDATE()` 조건 필수 포함

### 히어로배너 썸네일 에디터 스타일 깨짐
- **원인**: 어드민 페이지 CSS와 프론트 CSS 충돌
- **해결**: 에디터 영역을 고유 클래스(`.hero-editor-wrap`)로 감싸서 스코프 격리

### 플로팅 패널에 빈 카드 표시
- **원인**: 추천업소 라벨은 있지만 jr_data가 불완전
- **해결**: 렌더링 시 필수 필드(nickname, title) 없으면 스킵
