# 🚨 [중요] 기업회원 가입 시스템 전환 액션플랜

> **우선순위: 높음**
> **상태: 대기 (사업자등록 완료 후 진행)**
> **선행조건: 사업자등록 완료 → data.go.kr API 키 발급**

---

## 📌 개요

기업회원 가입 시 **Gemini AI OCR 스캔**을 제거하고, **국세청 사업자등록정보 진위확인 API**로 전환한다.
이를 통해 비용 절감, 정확도 향상, 자동 승인 프로세스를 구현한다.

---

## 📌 선행 조건 (반드시 먼저)

| 순서 | 작업 | 상태 |
|------|------|------|
| 1 | 사업자등록 완료 | ⏳ 대기 |
| 2 | [data.go.kr](https://www.data.go.kr/data/15081808/openapi.do) 회원가입 | ⏳ 대기 |
| 3 | "국세청_사업자등록정보 진위확인 및 상태조회 서비스" 활용신청 | ⏳ 대기 |
| 4 | 인코딩 키(Service Key) 발급 확인 (자동승인) | ⏳ 대기 |
| 5 | API 키를 서버 설정 파일에 등록 | ⏳ 대기 |

## 📌 현재 구조 (변경 전)

```
[기업회원 가입]
  ↓
  양식 작성 + 문서 첨부
  ↓
  [AI 자동인식] → Gemini Vision OCR (비용 발생, 정확도 80~95%)
  ↓
  가입 완료 (승인대기)
  ↓
  [어드민] AI 재스캔 + OCR 비교 + 수동 승인/반려
```

### 현재 문제점
- AI OCR 비용 이중 발생 (가입 시 + 어드민 재스캔)
- AI 추정값이라 100% 신뢰 불가
- 모든 기업회원을 어드민이 수동 확인해야 함
- 폐업/휴업 사업자도 가입 가능

## 📌 새로운 구조 (변경 후)

### 1차 승인: 국세청 진위확인 (가입 시점 - 자동)

```
[기업회원 양식 작성]
  사업자등록번호, 개업일자, 상호, 대표자, 주소, 주업태, 주종목, 확인문서첨부
           ↓
     [조회] 버튼 클릭
           ↓
     국세청 API 진위확인 호출 (상태조회 + 진위확인)
           ↓
  ┌─────────────────────────────────┐
  │ "확인됨" + 계속사업자           │ → ✅ "조회가 확인되었습니다."
  │                                 │ → 가입 버튼 활성화
  ├─────────────────────────────────┤
  │ "확인불가"                      │ → ❌ "가입이 불가합니다."
  │                                 │ → 정보 수정 후 재조회 가능
  ├─────────────────────────────────┤
  │ 휴업자 / 폐업자                 │ → ❌ "휴업/폐업 사업자입니다."
  │                                 │ → 정보 수정 후 재조회 가능
  └─────────────────────────────────┘
```

**핵심**: "가입 차단"이 아니라, 정보를 수정하고 다시 [조회] 가능. 가입 버튼은 조회 확인 시에만 활성화.

### 2차 승인: 업태/종목 허용 목록 매칭 (가입 직후 - 자동/수동)

```
[1차 승인 완료 = 가입 완료]
           ↓
  사용자가 입력한 업태/종목 vs "허용 업태/종목 관리" 목록 비교
           ↓
  ┌──────────────────────────────┐
  │ 업태 또는 종목이 허용 목록에  │ → ✅ 2차 자동 승인 (approved)
  │ 포함됨 (부분 일치)           │ → 즉시 채용공고 등록 가능
  ├──────────────────────────────┤
  │ 허용 목록에 없음             │ → ⏸ 보류 (held)
  │                              │ → 어드민이 수동으로 판단
  │                              │ → 로그인은 가능, 채용공고 등록 불가
  └──────────────────────────────┘
```

## 📌 회원 상태값 (mb_7)

| mb_7 값 | 의미 | 로그인 | 채용공고 등록 |
|---------|------|--------|-------------|
| `verified` | 1차 승인 완료 (2차 자동 매칭 직전 상태) | ✅ | ❌ |
| `approved` | 1차 + 2차 모두 승인 완료 | ✅ | ✅ |
| `held` | 1차 승인 + 2차 보류 (업태/종목 불일치) | ✅ | ❌ |
| `rejected` | 어드민이 반려 | ❌ | ❌ |

### 상태 전이 흐름
```
가입 불가 (진위확인 실패) → 재시도 가능

가입 성공 → verified
  → 업태/종목 일치 → approved (자동)
  → 업태/종목 불일치 → held (보류)
    → 어드민 승인 → approved
    → 어드민 반려 → rejected
```

## 📌 어드민 승인관리 화면

### 필터 탭
| 탭 | 대상 | 어드민 액션 |
|----|------|-----------|
| 전체 | 모든 기업회원 | — |
| 보류 (`held`) | 업태/종목 불일치 | **승인 / 반려** |
| 승인완료 (`approved`) | 모든 검증 통과 | — |
| 반려 (`rejected`) | 반려된 회원 | 재승인 가능 |

### 어드민 상세보기 표시 내용
- 기본 회원정보 (아이디, 이름, 이메일, 연락처)
- 사업자 정보 (사업자번호, 상호, 대표자, 주소, 개업일자)
- **업태/종목** + 허용 목록 일치 여부 (✅ 승인가능 / ❌ 미등록)
- 국세청 진위확인 결과 (확인됨/확인불가, 사업자상태)
- 첨부 문서 이미지 (육안 확인용)

## 📌 국세청 API 정보

- **API명**: 국세청_사업자등록정보 진위확인 및 상태조회 서비스
- **URL**: https://www.data.go.kr/data/15081808/openapi.do
- **유형**: REST API
- **포맷**: JSON + XML
- **비용**: 무료
- **호출 제한**: 1회 100건, 1일 100만건
- **심의**: 자동승인

### 상태조회 API
- **용도**: 사업자번호만으로 영업 상태 확인
- **입력**: 사업자등록번호 (b_no)
- **반환**: 납세자상태 (계속사업자/휴업자/폐업자), 과세유형

### 진위확인 API
- **용도**: 사용자 입력 정보가 국세청 기록과 일치하는지 검증
- **입력**: 사업자번호, 개업일자, 대표자명, 상호, 주업태(b_sector), 주종목(b_type)
- **반환**: 일치 여부 (valid: "01" 확인됨 / "02" 확인불가)

## 📌 실수 방지 및 트러블슈팅

### 1. API 키 관련
- **문제**: API 키가 잘못되거나 만료됨
- **해결**: `extend/nts_config.php`에서 키 확인, data.go.kr 마이페이지에서 재발급

### 2. 진위확인 "확인불가" 반복
- **원인**: 개업일자 형식 오류 (YYYYMMDD 필수), 상호/대표자 띄어쓰기 차이
- **해결**: 입력값 정규화 (공백 제거, 날짜 포맷 통일)

### 3. 자동승인이 안 되는 경우
- **원인**: "허용 업태/종목 관리"에 해당 업태/종목이 등록 안 됨
- **해결**: 어드민 → "허용 업태/종목 관리" 페이지에서 추가

### 4. 이미 가입한 기업회원 (기존 데이터)
- **문제**: 기존 `pending` 상태 회원은 국세청 진위확인을 거치지 않았음
- **해결**: 어드민에서 기존 회원은 수동으로 승인/반려 처리 (기존 방식 유지)
