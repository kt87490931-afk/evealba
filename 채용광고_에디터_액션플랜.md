### 1.4 수정 모달 — 리치 에디터 (굵기·컬러만)

**요구사항:**

- 수정 버튼 클릭 시 편집 영역에서 **굵기**와 **글자색**만 사용
- 회원이 드래그로 텍스트 선택 후 굵기·컬러 변경 가능

**구현 방식 — 방식A (contenteditable + execCommand) 권장:**

| 방식 | 설명 | 적합도 |
|------|------|--------|
| **방식A** | contenteditable + document.execCommand('bold', 'foreColor') | ✅ **가장 간단** — 별도 라이브러리 없음, 굵기·컬러만이면 충분 |
| 방식B | SmartEditor2 커스텀 스킨 (굵기·컬러만) | 가능하나 스킨 수정·분리 작업 필요 |
| 방식C | Quill/TinyMCE 등 외부 라이브러리 | 굵기·컬러만 위해 라이브러리 추가는 과함 |

→ 굵기·컬러만 필요 시 **방식A가 가장 간단**하고 경량임.

**방식A 상세 구현 플랜:**

| 단계 | 내용 | 비고 |
|------|------|------|
| 1 | 모달 내 textarea → `contenteditable="true"` div로 교체 (또는 수정 시 토글) | 편집 영역 |
| 2 | 굵기 버튼: 클릭 시 `document.execCommand('bold', false, null)` | 선택 영역에 적용 |
| 3 | 컬러 버튼: `<input type="color">` 또는 팔레트 → `document.execCommand('foreColor', false, '#hex')` | 선택 영역에 적용 |
| 4 | 저장 시: `contenteditable.innerHTML` 또는 `innerHTML`을 HTML 문자열로 수집 → disp 영역 반영 및 jr_data 저장 | `<strong>`, `<span style="color:...">` 등 |
| 5 | 로드 시: 저장된 HTML을 contenteditable div에 `innerHTML`로 주입 | jr_data → 편집 영역 |

- **드래그 선택** → 굵기/컬러 버튼 클릭 시 `execCommand`가 자동으로 선택 영역에 적용
- 별도 라이브러리 없음, 순수 HTML/JS만 사용

## Part 2: jr_data 로드 → 폼 input 자동 입력

### 2.1 목표

- 저장된 `jr_data`를 읽어와 에디터·폼의 각 input에 자동 입력
- jr_id 기준 조회 → 필드 매핑 → 폼 채우기

### 2.2 현재 준비 상태

| 항목 | 준비 여부 | 비고 |
|------|-----------|------|
| jr_data → input 매핑 정보 (키 이름) | ✅ 준비됨 | job_nickname, job_title, desc_location 등 |
| jr_id로 jr_data 조회 | ❌ 없음 | API 또는 페이지 로직 필요 |
| 폼에 jr_data 로드 | ❌ 없음 | PHP value 출력 또는 JS fetch 후 채우기 |
| 편의사항/키워드/MBTI 체크박스 복원 | ❌ 없음 | 배열 → 체크박스 복원 |
| AI 섹션 필드(ai_intro 등) 매핑 | ❌ 없음 | inp-recruit, inp-location 등 |

### 2.3 액션 항목

| 순번 | 액션 | 파일 | 산출물 |
|------|------|------|--------|
| 2.1 | jr_id로 jr_data 조회 API 또는 페이지 생성 | 새 API 또는 기존 파일 확장 | jr_data JSON 반환 |
| 2.2 | 로드 시 jr_data fetch 후 disp 영역·모달 input 채우기 | theme/evealba/ 또는 에디터 JS | 폼 자동 로드 |
| 2.3 | 편의사항·키워드·MBTI 체크박스 복원 (selectedBenefits 등) | JS | 체크박스 복원 |
| 2.4 | 기본 필드 매핑 (닉네임, 연락처, SNS, 급여, 근무지역, 업종 등) | PHP/JS | 기본 필드 로드 |
| 2.5 | AI 섹션 매핑 (ai_intro, ai_location, ai_env, ai_benefit, ai_wrapup) | PHP/JS | AI 필드 로드 |
| 2.6 | 저장 시 jr_data 업데이트 연동 (기존 jobs_ai_section_save 등 활용) | jobs_ai_section_save.php 등 | 수정 → DB 반영 |
