# 에이전트 핸드오프 문서 — 프리미엄채용정보 모바일 3열

**작성일**: 2026-02-27  
**목적**: 다른 에이전트가 즉시 작업을 이어갈 수 있도록 진행상황 정리

---

## 1. 작업 목표

모바일에서 **프리미엄채용정보**가 **가로 3개씩(3열)**로 표시되어야 하는데, 현재 **2열**로 나오는 문제를 수정 후 배포.

---

## 2. 이미 완료된 작업

### 2.1 카드 구조 통일
- **파일**: `theme/evealba/index_main.php`
- **내용**: 프리미엄/스페셜채용정보 카드를 `render_premium_card` → `render_job_card`로 변경
- **결과**: 우대채용정보와 동일하게 지역, 위치, 광고제목, 급여, 좋아요 표시

### 2.2 모바일 프리미엄 3열 CSS 수정 (1차 시도)
- **파일**: `theme/evealba/css/evealba.css`
- **내용**: `@media (max-width:600px)` 내 `.premium-grid`를 `repeat(2,1fr)` → `repeat(3,1fr)`로 변경

---

## 3. 남은 문제 (미해결)

- 모바일에서 **프리미엄채용정보가 여전히 2열**로 표시됨
- 3열로 수정했다고 가정했으나 실제 화면에는 반영되지 않음

---

## 4. 확인할 사항 및 수정 방향

### 4.1 현재 CSS 상태 (evealba.css)
- **기본**: `.premium-grid { grid-template-columns:repeat(5,1fr); }` (1216행)
- **미디어쿼리**:
  - 1513~1515행, 1620~1622행, 1648~1650행: `repeat(3,1fr) !important` 적용됨
  - 2322~2324행: `@media (max-width:768px)`에서 3열 강제 적용됨
- evealba.css에는 이미 3열 규칙이 다수 있으나, **실제 화면에서는 2열로 표시**됨

### 4.2 의심 구간
- **eve_alba_main.html** 1016행: `.premium-grid { grid-template-columns:repeat(2,1fr); gap:8px; }` — 이 파일이 실제로 로드되는지 확인 필요
- 모바일 전용 CSS나 인라인 스타일에서 2열로 덮어쓰는지 확인

### 4.3 수정 방향
1. evealba.css에서 `.premium-grid` 2열 규칙이 남아 있는지 전체 검색
2. 더 높은 specificity로 3열 강제 (예: `body .main-area .section-wrap .premium-grid`)
3. eve_alba_main.html이 로드된다면 해당 파일 내 2열 규칙도 3열로 수정

---

## 5. 배포 방법

수정 후 아래 명령 실행:

```powershell
cd d:\evealba
.\deploy.ps1 -Message "모바일 프리미엄채용정보 3열 수정" -PushToServer
```

## 7. 참고: special-grid (3열 동작)

`.special-grid`는 이미 모바일에서 3열로 설정되어 있음. 이 스타일을 참고하여 `.premium-grid`에도 동일하게 적용.

---

## 8. 실행 요약 (다른 에이전트용)

1. `theme/evealba/css/evealba.css`에서 `.premium-grid` 관련 모든 규칙 검색
2. 모바일 미디어쿼리(`max-width:600px`, `max-width:900px` 등) 내부에서 `repeat(2,1fr)` → `repeat(3,1fr)`로 변경 (필요 시 `!important` 추가)
3. `.\deploy.ps1 -Message "모바일 프리미엄채용정보 3열 수정" -PushToServer` 실행
4. 서버 반영 및 화면 확인

---

## 9. 참고: 모바일 페이지 구조

- `theme/evealba/index.php`: G5_IS_MOBILE이면 `mobile/index.php` 로드 (최신글만 표시, index_main 없음)
- PC/반응형(뷰포트만 좁음): `index_main.php` 로드 → 프리미엄/스페셜 섹션 표시
- 사용자가 모바일에서 프리미엄 2열을 본다면, 반응형 모드이거나 모바일에서도 index_main을 쓰는 별도 설정일 수 있음
