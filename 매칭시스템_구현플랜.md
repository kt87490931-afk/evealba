# 매칭시스템 구현플랜

## 1. 개요

### 1.1 목적
기업회원과 이브회원을 **매일 1쌍씩** AI 기반으로 자동 매칭하고, 매칭 시 쪽지로 알림·링크를 발송하는 시스템 구축.

### 1.2 요구사항 요약

| 항목 | 내용 |
|------|------|
| 매칭 단위 | 회원별 매일 1쌍 |
| 재매칭 | 1주일 전 매칭 이력 있어도 재매칭 가능 |
| 매칭 실패 | 후보 없으면 매칭하지 않음 |
| 알림 | 쪽지 (추후 앱에서는 푸시 알림 연동 예정) |
| 관리 | 어드민 > 이브알바전용 > 매칭시스템 |
| 운영 시점 | 기업·이브회원 가입이 충분할 때 |

### 1.3 매칭 조건

| 구분 | 조건 |
|------|------|
| **필수 일치** | 업종/직종 동일, 근무지역 1순위 동일 |
| **일치율** | 70% 이상 시 매칭 |
| **비교 항목** | 고용형태, 급여조건, 업종, 직종, 편의사항, 키워드, 선호MBTI, 업소위치, 근무환경, 지역혜택·복리후생, 지원자격·우대사항 |

## 3. Phase 0: 데이터 사전조사

### 3.1 기업회원 데이터 (AI소개글 종합정리)

| 출처 | 테이블/컬럼 | 비고 |
|------|-------------|------|
| AI 요약 | `g5_jobs_ai_content` (ai_data JSON) | ai_intro, ai_location, ai_env, ai_welfare, ai_qualify, ai_extra, ai_mbti_comment |
| 기본정보 | `g5_jobs_register` (jr_data, 개별 컬럼) | 고용형태, 급여조건, 업종(job1/job2), 근무지역(region 등), 편의사항, 키워드, MBTI |
| 활성 여부 | `g5_jobs_register` (jr_status, jr_end_date) | ongoing·결제기간 내만 매칭 대상 |

### 3.2 이브회원 데이터 (AI 매칭 이력서)

| 출처 | 테이블/컬럼 | 비고 |
|------|-------------|------|
| 이력서 | `g5_resume` | rs_data(LONGTEXT), rs_job1, rs_job2, rs_region, rs_work_region, rs_salary_type 등 |
| **필수** | "AI 매칭 이력서" 전용 저장소 | 현재 구조 확인 후, 없으면 신규 테이블/컬럼 추가 필요 |

### 3.3 액션

1. `g5_resume`의 rs_data JSON 구조 및 매칭용 필드 유무 확인
2. 이브회원용 "AI 매칭 이력서"가 별도 저장되는지 여부 확인
3. 없을 경우 `g5_resume_ai_match` 또는 rs_data 확장 방안 설계

## 4. Phase 1: 매칭용 DB 스키마

### 4.1 신규 테이블: g5_ev_matching_log

```sql
CREATE TABLE IF NOT EXISTS g5_ev_matching_log (
  mlog_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mb_id_eve VARCHAR(20) NOT NULL COMMENT '이브회원 mb_id',
  mb_id_ent VARCHAR(20) NOT NULL COMMENT '기업회원 mb_id',
  jr_id INT UNSIGNED NOT NULL COMMENT '채용정보 ID',
  rs_id INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '이력서 ID',
  match_rate TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '일치율 0~100',
  matched_at DATETIME NOT NULL COMMENT '매칭 일시',
  memo_sent TINYINT(1) NOT NULL DEFAULT 0 COMMENT '쪽지 발송 여부',
  KEY idx_mb_eve (mb_id_eve),
  KEY idx_mb_ent (mb_id_ent),
  KEY idx_matched_at (matched_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='매칭 이력';
```

### 4.2 신규 테이블: g5_ev_matching_config

```sql
CREATE TABLE IF NOT EXISTS g5_ev_matching_config (
  mc_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mc_key VARCHAR(50) NOT NULL UNIQUE,
  mc_value TEXT,
  mc_updated DATETIME,
  KEY idx_key (mc_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='매칭 시스템 설정';

-- 초기값
INSERT INTO g5_ev_matching_config (mc_key, mc_value, mc_updated) VALUES
('enabled', '0', NOW()),
('min_rate', '70', NOW()),
('re_match_days', '7', NOW()),
('min_eve_count', '10', NOW()),
('min_ent_count', '5', NOW());
```

### 4.3 이브회원 매칭용 데이터 확보

- **A안**: `g5_resume`에 rs_ai_match_data(JSON) 컬럼 추가 → AI 매칭 이력서 저장
- **B안**: 별도 테이블 `g5_resume_ai_match` 생성 (rs_id, mb_id, match_data JSON, created_at)

## 5. Phase 2: 매칭 로직 핵심

### 5.1 lib/ev_matching.lib.php 구조

```
ev_matching_run()           # 메인 진입점
  ├─ ev_matching_config_get()
  ├─ ev_matching_candidates_eve()  # 이브회원 후보 (AI 매칭 이력서 있는 자)
  ├─ ev_matching_candidates_ent()  # 기업회원 후보 (진행중 광고 있는 자)
  ├─ ev_matching_filter_mandatory() # 업종/직종, 1순위 지역 필터
  ├─ ev_matching_calc_rate()       # 일치율 산출
  └─ ev_matching_pair_one_day()    # 회원별 1쌍 선정 (7일 이내 재매칭 허용)
```

### 5.2 일치율 산출 규칙

| 항목 | 가중치 | 비교 방식 |
|------|--------|----------|
| 고용형태 | 10% | 문자열/코드 일치 |
| 급여조건 | 10% | 범위/유형 매핑 후 비교 |
| 업종/직종 | 15% | **필수 일치** (미일치 시 후보 제외) |
| 편의사항 | 10% | 교집합 비율 |
| 키워드 | 10% | 교집합 비율 |
| 선호MBTI | 5% | 그룹(NT/NF/SJ/SP) 또는 완전일치 |
| 업소위치 | 10% | 지역 코드/이름 정규화 후 비교 |
| 근무환경 | 10% | 키워드 유사도 |
| 지역혜택·복리후생 | 10% | 키워드 유사도 |
| 지원자격·우대사항 | 10% | 키워드 유사도 |
| **1순위 근무지역** | — | **필수 일치** (미일치 시 후보 제외) |

### 5.3 재매칭 로직

- `g5_ev_matching_log`에서 `mb_id_eve`, `mb_id_ent` 조합의 `matched_at` 조회
- `matched_at`이 7일 이내면 해당 쌍은 후보에서 제외 (또는 가중치 감점)
- 7일 초과 시 재매칭 허용

## 6. Phase 3: 매칭 실행·쪽지 발송

### 6.1 Cron 스크립트: cron_matching.php

```
1. config.enabled = 1 확인
2. 최소 회원 수(config) 확인
3. ev_matching_run() 호출
4. 매칭된 각 쌍에 대해:
   - g5_ev_matching_log INSERT
   - ev_send_memo() 2회 호출 (이브→광고링크, 기업→이력서링크)
5. 로그 기록 (매칭 건수, 실패 사유 등)
```

### 6.2 쪽지 템플릿

**이브회원 수신:**
```
지금 확인하세요! {M/D} 기준 매칭률 {rate}% 돌파 중 🚀

[{기업닉네임}]님의 채용정보와 {rate}% 매칭되었어요.
아래 버튼에서 자세히 확인해 보세요.

[채용정보 보기] → jobs_view.php?jr_id={jr_id}
```

**기업회원 수신:**
```
지금 확인하세요! {M/D} 기준 매칭률 {rate}% 돌파 중 🚀

[{이브닉네임}]님의 이력서와 {rate}% 매칭되었어요.
아래 버튼에서 자세히 확인해 보세요.

[이력서 보기] → talent_view.php?rs_id={rs_id} (또는 해당 URL)
```

- 쪽지 본문에 HTML `<a href="...">` 형태로 버튼 스타일 링크 포함 가능 (그누보드 쪽지 뷰어 지원 범위 확인)

## 10. 동일 실수 발생 시 대응

### 10.1 매칭이 전혀 되지 않을 때

| 원인 | 확인 방법 | 조치 |
|------|----------|------|
| config.enabled=0 | g5_ev_matching_config 조회 | enabled=1로 변경 |
| AI 매칭 이력서 없음 | g5_resume / g5_resume_ai_match 확인 | 이력서 작성·AI 저장 플로우 점검 |
| AI소개글 없음 | g5_jobs_ai_content, jr_data 확인 | 채용정보 AI 생성 여부 확인 |
| 필수 조건 과다 | 업종/직종, 1순위 지역 코드 비교 로그 | 비교 로직 완화 또는 데이터 정규화 |
| 최소 인원 미달 | min_eve_count, min_ent_count | 설정 조정 또는 임시 비활성 |

### 10.2 같은 사람만 반복 매칭될 때

| 원인 | 확인 방법 | 조치 |
|------|----------|------|
| 재매칭 7일 체크 누락 | ev_matching_pair_one_day 로직 | matched_at 기준 7일 필터 확인 |
| 후보 pool 부족 | 이브/기업 후보 수 로그 | 가입 유도 또는 재매칭 기간 연장 |

### 10.3 쪽지가 가지 않을 때

| 원인 | 확인 방법 | 조치 |
|------|----------|------|
| ev_send_memo 실패 | PHP 에러 로그, 반환값 | mb_id 유효성, 쪽지 테이블 권한 확인 |
| memo_sent 미갱신 | g5_ev_matching_log | INSERT 후 memo_sent=1 업데이트 확인 |
| 쪽지 용량 초과 | me_memo 길이 | 내용 축소 또는 분할 발송 |

### 10.4 일치율이 비현실적으로 높/낮을 때

| 원인 | 확인 방법 | 조치 |
|------|----------|------|
| 가중치 불균형 | ev_matching_calc_rate 단계별 로그 | 가중치 재조정 |
| 비교 필드 누락 | ai_data, rs_data 키 매핑 | 누락 필드 추가 |
| 문자열 정규화 부족 | "서울" vs "서울시" 등 | 전처리 함수 추가 (지역·업종 코드화) |
