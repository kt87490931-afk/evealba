# 점프 기능 구현 액션플랜 (최종)

## 1. 개요

점프는 광고주가 자신의 채용공고를 해당 리스트 최상단으로 올리는 기능입니다.
줄광고 필수 결제 시 기본 점프 횟수가 부여되며, 추가 구매도 가능합니다.

## 2. 점프 횟수 부여 체계

### 줄광고 단독 결제 시

| 기간 | 점프 횟수 | 금액 |
|------|----------|------|
| 30일 | 300회 | 15,000원 |
| 60일 | 700회 | 30,000원 |
| 90일 | 1,200회 | 50,000원 |

### 줄광고 + 다른광고 (우대/프리미엄/스페셜/급구/추천) 초과 결제 시

## 3. 점프 동작 규칙

### 3-1. 다중 리스트 동시 점프 (1회 차감)

점프 1회 실행 시 결제한 모든 광고 유형 리스트에서 동시에 최상단으로 이동:
- 줄광고+우대 → 줄광고 리스트 최상단 + 우대 리스트 최앞
- 줄광고+급구 → 줄광고 리스트 최상단 + 급구 리스트 최앞 + 마키 배너 최앞
- 특수배너는 히어로 영역이므로 점프 대상 아님

### 3-2. 수동 점프

버튼 클릭 → 즉시 점프 → 잔여횟수 -1

### 3-3. 자동 점프

- 남은 기간과 잔여 점프 횟수를 기반으로 균등 간격 자동 계산
- 예: 남은 17일, 잔여 170회 → 하루 10회 → 약 144분(2시간 24분)마다 1회
- 랜덤 오프셋 (±5분): 300명이 동시 자동점프 시 서버 부하 방지 + 공정성 확보
- 잔여 횟수 0이 되면 자동 중지
- 수동 점프 사용 시 자동 간격 재계산 (잔여횟수 감소 반영)
- 추가 구매 시 자동 간격 재계산 (잔여횟수 증가 반영)

### 3-4. 정렬 기준

```sql
ORDER BY IFNULL(jr_jump_datetime, jr_datetime) DESC
```

## 4. DB 스키마

### 4-1. g5_jobs_register 컬럼 추가

```sql
ALTER TABLE g5_jobs_register
  ADD COLUMN jr_jump_remain INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '잔여 점프 횟수',
  ADD COLUMN jr_jump_used INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '사용한 점프 횟수',
  ADD COLUMN jr_jump_total INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '총 부여 점프 횟수',
  ADD COLUMN jr_jump_datetime DATETIME DEFAULT NULL COMMENT '마지막 점프 시각',
  ADD COLUMN jr_auto_jump TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '자동점프 활성화 (0=OFF, 1=ON)',
  ADD COLUMN jr_auto_jump_next DATETIME DEFAULT NULL COMMENT '다음 자동 점프 예정 시각',
  ADD INDEX idx_jump_datetime (jr_jump_datetime),
  ADD INDEX idx_auto_jump_next (jr_auto_jump, jr_auto_jump_next);
```

### 4-2. 점프 이력 테이블

```sql
CREATE TABLE IF NOT EXISTS g5_jobs_jump_log (
  jl_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  jr_id INT UNSIGNED NOT NULL,
  mb_id VARCHAR(20) NOT NULL DEFAULT '',
  jl_type ENUM('manual','auto') NOT NULL DEFAULT 'manual',
  jl_remain_before INT UNSIGNED NOT NULL DEFAULT 0,
  jl_remain_after INT UNSIGNED NOT NULL DEFAULT 0,
  jl_datetime DATETIME NOT NULL,
  KEY idx_jr_id (jr_id),
  KEY idx_datetime (jl_datetime)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4-3. 점프 추가 구매 테이블

```sql
CREATE TABLE IF NOT EXISTS g5_jobs_jump_purchase (
  jp_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  jr_id INT UNSIGNED NOT NULL,
  mb_id VARCHAR(20) NOT NULL DEFAULT '',
  jp_count INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '구매 횟수',
  jp_amount INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '결제 금액',
  jp_status VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT 'pending/confirmed',

  KEY idx_mb_id (mb_id)

```

## 6. 자동 점프 간격 계산 공식

```
남은일수 = jr_end_date - TODAY
잔여횟수 = jr_jump_remain
남은분 = 남은일수 × 24 × 60
간격(분) = FLOOR(남은분 / 잔여횟수)
최소간격 = MAX(간격, 10) (최소 10분)
랜덤오프셋 = RAND(-5, +5)분
다음점프시각 = NOW() + 간격 + 랜덤오프셋
```

## 7. 결제 확인 시 점프 횟수 자동 지급

어드민에서 `jr_payment_confirmed = 1` 설정 시:
1. `jr_ad_labels` 파싱하여 광고 유형 확인
2. `jr_ad_period`로 기간 확인 (30/60/90일)
3. 줄광고만 vs 줄광고+다른광고 구분
4. 해당 점프 횟수를 `jr_jump_total`, `jr_jump_remain`에 자동 부여

## 8. 실수했을 때의 해결책

### 점프 횟수 불일치
- **확인**: `jr_jump_total = jr_jump_used + jr_jump_remain`이 항상 성립
- **해결**: 이력 테이블 합계와 대조하여 보정

### 자동 점프 간격이 너무 짧음
- **원인**: 잔여횟수 대비 남은 기간이 너무 김
- **해결**: 최소 간격 10분 강제 적용

### 수동+자동 동시 실행 시 중복 차감
- **해결**: 점프 실행 시 트랜잭션으로 `jr_jump_remain > 0` 확인 후 차감

### 추가 구매 후 자동 간격 미갱신
- **해결**: 구매 확인 시 자동으로 `jr_auto_jump_next` 재계산
