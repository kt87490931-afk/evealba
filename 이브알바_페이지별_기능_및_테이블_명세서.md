## 2. 1차: 기반 테이블 및 공통 기능

> **목표**: 로그인·회원·마스터코드가 동작하는 최소 기반

### 2.1 1차-1: Gnuboard 기본 테이블 (이미 존재)

| 테이블 | 용도 | 참고 |
|--------|------|------|
| g5_config | 사이트 설정 (제목, 전화번호, 운영시간 등) | 고객센터 전화번호 등 |
| g5_member | 회원 (mb_id, mb_name, mb_level, mb_type 등) | 로그인·이력서·채용공고 주체 |
| g5_member_extra | 회원 확장 (Gnuboard 옵션) | 업소/구직자 구분 등 |
| g5_board | 게시판 설정 (bo_table, bo_subject 등) | 수다방·고객센터 게시판 |
| g5_write_{bo_table} | 게시글 (wr_id, wr_subject, wr_content 등) | 동적 생성 |
| g5_content | 정적 페이지 (이용약관, 개인정보처리방침) | content.php |
| g5_visit | 방문자 통계 | 오늘 접속자 카운트 |
| g5_login | 로그인 기록 | - |

### 2.2 1차-2: 이브알바 마스터 테이블 (migrations/001 존재)

| 테이블 | 용도 | 주요 컬럼 |
|--------|------|----------|
| g5_ev_industry | 1차 업종 (룸싸롱, 노래주점, 마사지, 기타) | ei_id, ei_code, ei_name |
| g5_ev_job | 2차 직종 (아가씨, 초미씨, 미씨, TC) | ej_id, ei_id, ej_code, ej_name |
| g5_ev_region | 1차 지역 (서울, 경기, 인천 등) | er_id, er_code, er_name |
| g5_ev_region_detail | 2차 세부지역 (강남구, 서초구 등) | erd_id, er_id, erd_code, erd_name |
| g5_ev_convenience | 편의사항 (선불가능, 순번확실 등) | ec_id, ec_code, ec_name |
| g5_ev_keyword | 키워드 마스터 | ek_id, ek_code, ek_name |
| g5_ev_mbti | MBTI 마스터 | em_id, em_code, em_name |

### 2.3 1차-3: 공통 기능 (필수 구현)

| 기능 | 설명 | 테이블 |
|------|------|--------|
| 로그인 폼 연동 | 사이드바 로그인 → Gnuboard login.php | g5_member |
| 회원 유형 구분 | 업소회원 vs 구직자회원 | g5_member.mb_1~10 또는 g5_member_extra |
| 오늘 방문자 수 | g5_visit 기반 실시간 카운트 | g5_visit |
| 지역/직종 셀렉트 옵션 | filter select에 마스터 데이터 주입 | g5_ev_region, g5_ev_region_detail, g5_ev_industry, g5_ev_job |
| 검색 폼 action | 헤더 검색 → /bbs/search.php | g5_board, g5_write_* |

### 2.4 1차 구현 순서

1. `g5_ev_*` 마스터 데이터 INSERT (지역, 직종, 편의, 키워드 시드)
2. jobs.php, talent.php filter 셀렉트를 DB 조회로 교체
3. 사이드바 로그인 → Gnuboard 기본 폼 연동
4. g5_visit 기반 “오늘 방문 N명” 표시

## 3. 2차: 채용정보·인재정보 핵심 CRUD

> **목표**: 채용공고·이력서 등록·조회·수정·삭제

### 3.1 2차-1: 채용공고 테이블 (신규)

```
g5_ev_job_post (채용공고)
├── jp_id (PK)
├── mb_id (등록자, 업소회원)
├── er_id (지역 FK)
├── erd_id (세부지역 FK, nullable)
├── ei_id (1차 업종 FK)
├── ej_id (2차 직종 FK, nullable)
├── jp_title (채용제목)
├── jp_content (상세내용)
├── jp_gender (성별: 여/남/무관)
├── jp_age_min, jp_age_max (연령)
├── jp_wage_type (급여유형: 일불/시간/월불/협의 등)
├── jp_wage_min, jp_wage_max (급여 범위)
├── jp_convenience (편의사항 JSON 또는 별도 매핑테이블)
├── jp_ad_type (광고유형: 일반/우대/프리미엄/스페셜/급구/추천)
├── jp_ad_expire (광고만료일)
├── jp_status (상태: 등록/마감/삭제)
├── jp_hit (조회수)
├── jp_datetime, jp_updated
└── jp_ip
```

### 3.2 2차-2: 이력서(인재정보) 테이블 (신규)

```
g5_ev_resume (이력서/인재정보)
├── re_id (PK)
├── mb_id (등록자, 구직자회원)
├── er_id (희망지역 FK)
├── ei_id (희망업종 FK)
├── ej_id (희망직종 FK, nullable)
├── re_title (제목)
├── re_content (상세내용)
├── re_name_display (노출이름, 마○○ 형태)
├── re_gender (성별)
├── re_age (나이)
├── re_wage_type (희망급여유형)
├── re_wage_amount (희망급여)
├── re_status (상태: 공개/비공개/삭제)
├── re_hit (조회수)
├── re_datetime, re_updated
└── re_ip
```

### 3.3 2차-3: 채용정보 페이지별 기능

| 영역 | 현재 | 필요 기능 | 테이블 |
|------|------|----------|--------|
| 메인배너 | 정적 | 클릭 시 연동 URL 또는 광고테이블 | g5_ev_ad_banner (옵션) |
| 검색 필터 | 정적 select | 지역·직종·편의·키워드로 검색 | g5_ev_job_post, g5_ev_* |
| 우대채용정보 | 정적 카드 | jp_ad_type=우대 목록 조회 | g5_ev_job_post |
| 프리미엄/스페셜 | 정적 | jp_ad_type별 목록 조회 | g5_ev_job_post |
| 급구/추천 | 정적 | jp_ad_type, 추천플래그 조회 | g5_ev_job_post |
| 채용정보 리스트 | 정적 테이블 | 페이징·정렬·필터 결과 | g5_ev_job_post |
| 채용공고 상세 | 미구현 | wr_id 또는 jp_id 상세 페이지 | g5_ev_job_post |
| 채용공고 등록 | 버튼만 | 로그인 후 등록 폼 | g5_ev_job_post, g5_member |

### 3.4 2차-4: 인재정보 페이지별 기능

| 영역 | 현재 | 필요 기능 | 테이블 |
|------|------|----------|--------|
| 업직종 탭 카운터 | 정적 | 직종별 이력서 COUNT | g5_ev_resume, g5_ev_industry |
| AI 매칭 배너 | 정적 | 클릭 시 매칭 페이지 이동 | - |
| 열람권 배너 | 정적 | 열람권 구매 페이지 연결 | g5_ev_view_ticket (4차) |
| 검색 필터 | 정적 | 지역·직종·키워드 검색 | g5_ev_resume |
| 인재정보 리스트 | 정적 테이블 | 페이징·필터·열람권 체크 | g5_ev_resume |
| 인재 상세 | 미구현 | 열람권 있을 때만 상세 노출 | g5_ev_resume, g5_ev_view_ticket |

### 3.5 2차 구현 순서

1. `g5_ev_job_post`, `g5_ev_resume` 테이블 생성 (migration)
2. 채용공고 등록 폼 페이지 (jobs_write.php 또는 별도)
3. 채용공고 목록 API/쿼리 (jobs.php, jobs_main.php 연동)
4. 이력서 등록 폼 페이지 (talent_write.php 또는 별도)
5. 인재정보 목록 API/쿼리 (talent.php, talent_main.php 연동)
6. 업직종별 COUNT 쿼리 (type-tab-bar)

## 4. 3차: 이브수다방·고객센터 게시판 연동

> **목표**: Gnuboard 게시판과 화면 1:1 매핑

### 4.1 3차-1: 게시판 설계 (g5_board + g5_write_*)

| 게시판(bo_table) | 용도 | 고유 설정 |
|------------------|------|----------|
| notice | 공지사항 (고객센터용) | 공지글 상단 고정 |
| qa | Q&A 문의 (광고·일반) | 비밀글, 답변글 |
| faq | FAQ | 카테고리 또는 wr_1 사용 |
| design | 디자인 문의 | 비밀글 |
| best | 베스트글 (수다방) | 좋아요 10개 이상 자동 선정 |
| night | 밤문화이야기 | 일반 게시판 |
| couple | 같이일할단짝찾기 | 일반 게시판 |
| legal | 무료법률자문 | 비밀글 가능 |
| used | 중고거래 | 이미지, 가격(wr_1 등) |

### 4.2 3차-2: 이브수다방 페이지별 기능

| 영역 | 현재 | 필요 기능 | 테이블 |
|------|------|----------|--------|
| 베스트글 | 정적 | wr_good>=10 정렬 상위 N건 | g5_write_best (또는 통합 bo) |
| 밤문화이야기 | 정적 | 최신글 목록 | g5_write_night |
| 같이일할단짝찾기 | 정적 | 최신글 목록 | g5_write_couple |
| 무료법률자문 | 정적 | 최신글 목록 | g5_write_legal |
| 중고거래 | 정적 카드 | 이미지·가격·지역 표시 | g5_write_used |
| 더보기/글쓰기 | 링크 # | get_pretty_url(bo_table) | g5_board |

### 4.3 3차-3: 고객센터 페이지별 기능

| 영역 | 현재 | 필요 기능 | 테이블 |
|------|------|----------|--------|
| 공지사항 | 정적 | notice 게시판 최신글 | g5_write_notice |
| FAQ | 정적 아코디언 | faq 게시판 조회 (Q:제목, A:내용) | g5_write_faq |
| Q&A 문의 | 정적 | qa 게시판 (원글+답글) | g5_write_qa |
| 디자인 문의 | 정적 | design 게시판 | g5_write_design |
| 더보기/문의하기 | 링크 # | get_pretty_url(bo_table) | g5_board |

### 4.4 3차-4: FAQ 구조

- **방식 A**: g5_write_faq 사용, wr_subject=질문, wr_content=답변
- **방식 B**: g5_ev_faq 전용 테이블 (faq_id, faq_question, faq_answer, faq_ord)

### 4.5 3차 구현 순서

1. g5_board에 notice, qa, faq, design, best, night, couple, legal, used 생성
2. g5_write_{bo_table} 자동 생성 (Gnuboard 기본)
3. sudabang_main.php → latest.lib.php 또는 직접 쿼리로 게시판 목록 연동
4. cs_main.php → 공지/FAQ/Q&A/디자인 게시판 연동
5. “더보기”, “글쓰기” 링크를 get_pretty_url(bo_table) 등으로 연결

## 5. 4차: 유료 기능·광고·열람권

|-------------|------|

| 열람권 구매 | 구직자 상세 열람 시 열람권 구매 → vt 생성 | g5_ev_view_ticket |
| 열람권 검증 | 인재 상세 페이지 진입 시 vt 유효 여부 체크 | g5_ev_view_ticket |

## 6. 5차: 부가 기능·통계·검색

> **목표**: 통계, 추천, 검색 고도화

### 6.1 5차-1: 메인 페이지 통계

| 영역 | 현재 | 필요 기능 | 테이블 |
|------|------|----------|--------|
| 오늘 채용공고 | 3,427 (정적) | COUNT(jp_id) WHERE DATE(jp_datetime)=TODAY | g5_ev_job_post |
| 등록 이력서 | 12,841 (정적) | COUNT(re_id) | g5_ev_resume |
| 가입 업소 | 8,920 (정적) | COUNT(mb_id) WHERE mb_type=업소 | g5_member |
| 오늘 접속자 | 24,153 (정적) | SUM(vi_count) WHERE vi_date=TODAY | g5_visit |
| 오늘 매칭 | 1,203 (정적) | 매칭 로그 테이블 또는 추정 로직 | g5_ev_match_log (옵션) |

### 6.2 5차-2: 티커 (상단 스크롤 텍스트)

| 페이지 | 티커 내용 | 데이터 소스 |
|--------|----------|------------|
| 채용정보 | 급구 채용 제목 | g5_ev_job_post (jp_ad_type=급구) |
| 인재정보 | 신규 이력서 제목 | g5_ev_resume (최신순) |

### 7.3 메뉴 추가 방법

- **자동 로드**: `adm/admin.lib.php`가 `admin.menu[0-9]{3}.*\.php` 패턴으로 `adm/` 폴더를 스캔하여 include. `adm/admin.menu600.php`를 생성하면 자동으로 메뉴에 반영됨.
- **파일 형식**: 기존 `admin.menu100.php` 등과 동일하게 `$menu['menu600'] = array(...)` 정의.
- **권한**: `auth_list.php`에서 `600000` ~ `600800` 권한 항목 추가 (super는 전체 접근).

**예시 `adm/admin.menu600.php`**:

```php
<?php
$menu['menu600'] = array(
    array('600000', '이브알바 관리', G5_ADMIN_URL . '/ev_dashboard.php', 'ev_dashboard'),
    array('600100', '대시보드', G5_ADMIN_URL . '/ev_dashboard.php', 'ev_dash'),
    array('600200', '마스터 관리', G5_ADMIN_URL . '/ev_master.php', 'ev_master'),
    array('600300', '채용공고 관리', G5_ADMIN_URL . '/ev_job_post_list.php', 'ev_job'),
    array('600400', '인재정보 관리', G5_ADMIN_URL . '/ev_resume_list.php', 'ev_resume'),
    array('600500', '광고·결제 관리', G5_ADMIN_URL . '/ev_ad_list.php', 'ev_ad'),
    array('600600', '열람권 관리', G5_ADMIN_URL . '/ev_view_ticket_list.php', 'ev_ticket'),
```
